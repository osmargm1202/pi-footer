# pi-footer v0.2.7

Real Zentui fork release for the ORGM Pi stack.

## Added

- Renamed fork package metadata/config/command to `pi-footer`.
- Documented ORGM stack install flow:

```bash
for pkg in pi-mem pi-caveman pi-harness pi-footer; do
  pi install git:github.com/osmargm1202/$pkg
done
```

- Added ORGM footer status line consuming existing stack state:
  - title display from `title:state-changed` / `session-title`
  - caveman status from `pi-caveman:state`
- Displays optional line like: `Build footer · caveman:full`.

## Fixed

- Delayed editor reconciliation now ignores stale Pi contexts after session replacement/reload.

## Notes

- `pi-footer` does not generate titles.
- Title generation remains in the ORGM stack outside this package.
- ChatGPT/Codex limits remain command-only in `pi-harness`, not persistent footer state.
