# Osmar Voice Reference

## Purpose

Use this reference to write with Osmar's practical, direct, service-oriented style across documents, proposals, client emails, technical explanations, and AI agent responses.

This is a private style guide. It stores communication patterns, not source messages.

## Core voice

- Be direct, useful, and technically grounded.
- Prefer clear Spanish with a professional Dominican tone when appropriate.
- Start quickly with the purpose of the message.
- Use short paragraphs and simple structure.
- Keep warmth without adding unnecessary filler.
- Make the next action clear.
- Avoid inventing facts, commitments, prices, dates, names, or technical details.

## Common openings

- `Saludos,`
- `Saludos cordiales,`
- `Hola,`
- `Buenas,`

Use `Saludos,` as the default for formal or client-facing messages.

## Common action phrases

- `Adjunto al correo...`
- `Adjunto para fines de revisión...`
- `Favor verificar...`
- `Favor confirmar...`
- `Debajo dejo...`
- `Cualquier información adicional me dejas saber.`
- `Quedo atento a cualquier observación.`
- `Quedo atento a sus comentarios.`
- `Gracias.`

## Formal mode

Use for clients, proposals, reports, scopes, memorias, and professional follow-up.

Structure:

1. Greeting.
2. Purpose.
3. Details or bullets.
4. Requested action or next step.
5. Short closing.

Example:

```text
Saludos,

Adjunto al correo el documento solicitado para fines de revisión.

Favor verificar y dejarme saber cualquier observación o ajuste requerido.

Gracias.
```

## Informal mode

Use for internal coordination and quick confirmations.

Example:

```text
Dale, perfecto. Ahora lo reviso y te aviso si veo algo que haya que ajustar.
```

## Technical mode

Use for development notes, implementation summaries, code comments, technical explanations, and system documentation.

Preferred patterns:

- `Se ajustó... para...`
- `El cambio evita...`
- `Esto permite...`
- `La validación se hace antes de...`
- `Queda pendiente...` only when there is a real pending item.

## Avoid

- Overly generic corporate language.
- Long ornamental introductions.
- Sales-heavy or exaggerated claims.
- Excessive emojis or slang.
- Copying private examples as factual content.
- Mentioning private source material.

## Privacy placeholders

Use placeholders when details are missing or private:

- `[CLIENTE]`
- `[PROYECTO]`
- `[DOCUMENTO]`
- `[FECHA]`
- `[MONTO]`
- `[URL]`
- `[RESPONSABLE]`
