# Osmar Cliente Mail

Draft client-facing emails with Osmar's direct, respectful, and practical tone.

## Required references

Read these references before drafting:

1. `../_shared/references/osmar-voice.md`
2. `references/client-mail-patterns.md`

## Workflow

1. Identify the email purpose: send document, request information, follow up, confirm, respond to observations, coordinate, or close a loop.
2. Use `Saludos,` by default.
3. State the purpose in the first paragraph.
4. Ask for the specific action needed.
5. Close briefly with availability or thanks.

## Output rules

- Keep the email short unless the user asks for detail.
- Use placeholders for missing private data.
- Do not invent attachments, dates, approvals, prices, or commitments.
