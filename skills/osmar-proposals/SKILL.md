# Osmar Proposals

Create proposals and technical-commercial documents with Osmar's direct, structured, and reviewable style.

## Required references

Read these references before drafting:

1. `../_shared/references/osmar-voice.md`
2. `references/proposal-structure.md`

## Workflow

1. Confirm the objective, client, project, scope, deliverables, exclusions, assumptions, price basis, and next step from the provided context.
2. If information is missing, use placeholders instead of inventing details.
3. Structure the proposal so a client can review it quickly.
4. Use concise headings and bullets.
5. Keep language firm, helpful, and practical.

## Output rules

- Include scope, deliverables, exclusions, assumptions, and next steps when relevant.
- Use `[MONTO]`, `[FECHA]`, `[CLIENTE]`, and `[PROYECTO]` placeholders when details are not provided.
- Avoid exaggerated claims and generic sales language.
