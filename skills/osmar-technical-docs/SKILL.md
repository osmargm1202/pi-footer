# Osmar Technical Docs

Create technical documents that are clear, structured, and practical, with Osmar's direct technical voice.

## Required references

Read these references before drafting:

1. `../_shared/references/osmar-voice.md`
2. `references/technical-doc-structure.md`

## Workflow

1. Identify the document type: memoria técnica, report, procedure, criteria, field note, system documentation, or explanation.
2. State the objective and context first.
3. Separate criteria, observations, procedure, results, and next steps.
4. Use bullets, tables, or numbered steps when they improve reviewability.
5. Keep technical claims tied to provided facts.

## Output rules

- Use placeholders when project details are missing.
- Prefer concise technical wording.
- Avoid decorative comments and unsupported conclusions.
